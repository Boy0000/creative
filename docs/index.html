Uracle

Why Uracle

There are a lot of plugins that use Minecraft resource-packs to extend the default Minecraft experience, using resource-packs allows servers to do a lot of this that would not be possible without them: adding new entities, items, emojis, inventories, music, etc.

That's where Uracle joins, Uracle is an API for those plugins, to make them work together, take it as "Vault" for next generation servers

Inspiration

Uracle is expected to be an essential API for these next-generation servers where a resource-pack is required, similar to Vault for economy and chat integration

Installation

Installation requires a few steps:

  1. Download the Uracle.jar file from GitHub

  2. Put the Uracle.jar file in your plugins/ folder in Spigot or Paper

  3. Start your server to generate the default configuration files

File Structure

The Uracle file structure is very simple, it contains a single configuration file: the config.yml file, two folders and an image (resource pack icon)

  • config.yml (file)

    In this file you can set resource-pack metadata (pack.mcmeta), resource-pack prompt and modify how to apply it (whether it's required or not)

  • pack.png (optional file)

    This is the resource-pack icon, shown in the resource-pack list in the Minecraft client

  • overrides (optional folder)

    Files that will be included in the final resource-pack, they can't be overridden by plugins when writing their own assets, be careful when including files here

  • optionals (optional folder)

    Files that will be included in the final resource-pack if they aren't overridden by plugins (recommended instead of overrides)

Configuration

This section describes how to edit the config.yml file depending on what you want to do

Change description or Minecraft version

You can change the resource-pack description (shown in resource-pack list in Minecraft client) and Minecraft version at the first section (metadata), e.g.


    metadata:
        format: 7
        description: "&6My Awesome Resource Pack"

Where description is the resource-pack description, accepts colors, and format specifies the Minecraft version, see the table below

Format Versions
1 1.6.1 - 1.8.9
2 1.9 - 1.10.2
3 1.11 - 1.12.2
4 1.13 - 1.14.4
5 1.15 - 1.16.1
6 1.16.2 - 1.16.5
7 1.17 - 1.17.1
8 1.18

Change generation

You can change how to generate the resource-pack, you decide if you want to upload it to a server like MCPacks or just generate a ZIP file or put all the files in a folder


    generation: mcpacks

There are some default values you can use, see the table below

Format Description Example
mcpacks Uploads the resource-pack to MCPacks, allows to dynamically apply resource-pack to players generation: mcpacks
zip Generates a ZIP file inside the plugin folder, the plugin won't apply the resource-pack to players if you use this option generation: zip
folder Generates a folder containing all the resource-pack assets inside the plugin folder, as in zip option, it won't apply the resource-pack to players. generation: folder

Application settings

You can configure if resource-pack must be forced (players can't join if they don't use the resource-pack) and configure the prompt message (shown when they are asked to use the resource-pack) in the application section:


    application:
        force: true
        prompt: "Hey! Please accept our &cresource-pack"

Where force determines if the resource-pack is required, and prompt specifies the prompt message (only shown if force is true)

That's all the configuration!

Writing Assets

Writing assets can't be simpler, it's just listening to ResourcePackGenerateEvent, in example:


    import team.unnamed.uracle.event.ResourcePackGenerateEvent;
    import team.unnamed.uracle.io.Writeable;

    public class PackGenerationListener implements Listener {

        @EventHandler
        public void onGenerate(ResourcePackGenerateEvent event) {

            // writing an image
            File icon = new File("/path/to/icon.png");
            event.write("pack.png", Writeable.ofFile(icon));

            // writing a credits.txt file
            event.write("credits.txt", "Credits");

            // writing raw bytes
            event.write("path/to/binary", new byte[0]);

            // writing a json element (Gson)
            JsonObject json = new JsonObject();
            json.addProperty("hello", "world");
            json.addProperty("foo", true);
            event.write("path/to/file", json);
        }

    }
